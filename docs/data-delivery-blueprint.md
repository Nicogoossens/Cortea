# Data Delivery Blueprint — Learning Track Questions

**Version:** 1.0 — April 2026
**Status:** Production reference
**Scope:** All countries. Belgium is the master template.

---

## Purpose

This document defines exactly how learning-track question data must be structured before it can be imported into the database via the batch seeder. Anyone delivering content for a new country (FR, DE, NL, UK, …) must follow this specification precisely. No custom code should be needed for a new country once the JSON file is correct.

---

## 1. End-to-end flow

```
Source material         Converter script              Seeder              Database
(Markdown / Excel)  →  scripts/import-XX-*.mjs  →  seed:XX (npm)  →  learning_track_questions
```

1. **Source material** — Raw content in any format (markdown, Excel, structured text).
2. **Converter script** — Country-specific Node.js `.mjs` script that parses the source and writes a JSON file to `data/<cc>-learning-tracks.json`.
3. **Seeder** — `pnpm --filter @workspace/db seed:be` (or `seed:XX` for another country) runs the existing `seed-learning-tracks.ts` against the generated JSON.
4. **Database** — Records land in `learning_track_questions` with a computed `question_hash` for idempotent re-runs.

---

## 2. JSON schema

The output file must be a **JSON array** of objects matching the schema below.

```jsonc
[
  {
    // ── Required ──────────────────────────────────────────────────────
    "register":      "middle-class",   // string — see § 3.1
    "phase":         1,                // integer 1–5 — see § 3.2
    "level":         1,                // integer 1–5 — see § 3.3
    "region_code":   "BE",             // ISO 3166-1 alpha-2, uppercase
    "demographic":   "general",        // string — see § 3.4
    "question_text": "…",              // string, non-empty
    "lang":          "nl",             // BCP 47 language tag — see § 3.5
    "options": [                       // array, 2–4 items
      {
        "text":        "…",            // string, non-empty
        "answer_tier": 1,              // integer 1 | 2 | 3 — see § 3.6
        "motivation":  "…"             // string, non-empty
      }
    ],

    // ── Optional ──────────────────────────────────────────────────────
    "research_pillar":    "adaptive_linguistics",  // string | null
    "historical_context": "…"                      // string | null
  }
]
```

Every object in the array maps directly to a row in `learning_track_questions`. The column `question_hash` is generated by the database and must **not** be included.

## 3. Field reference

### 3.1 `register`

The social class / register that the question content addresses.

| Value | Meaning |
|---|---|
| `middle-class` | Middle-class professional norms — the primary register |
| `professional` | Workplace and business-specific etiquette |
| `aristocracy` | Noble / high-society codes |

Use `middle-class` as the default for most questions. Only use `aristocracy` for content that is explicitly about noble circles.

---

### 3.2 `phase`

Which phase of the learning track this question belongs to, following the Blueprint rollout sequence.

| Phase | Scope | Audience |
|---|---|---|
| 1 | Individual Core | Everyone |
| 2 | Interaction Dynamics | Everyone |
| 3 | Social Navigation | Everyone |
| 4 | Country Layer (residents) | People living in the country |
| 5 | Contemporary Context (visitors/newcomers) | Visitors and newcomers |

Phases 1–3 are **universal** — the cultural references and examples change per country, but the structural pillars remain the same. Phases 4–5 are **country-specific**.

Belgium tip: the majority of the `Belgie_middenklasse` content is Phase 1. Regional content (Flanders vs. Wallonia vs. Brussels) belongs to Phase 4.

---

### 3.3 `level`

The difficulty / depth level within a phase.

| Level | Name | Description |
|---|---|---|
| 1 | Foundation | Entry-level rules; universally applicable |
| 2 | Social Arena | Group dynamics, public behaviour, dining |
| 3 | Material Modesty | Money, possessions, status signals |
| 4 | High-Stakes Diplomacy | Conflict, negotiation, authority |
| 5 | Crisis & Reputation | Scandal, burnout, legacy, high-pressure |

---

### 3.4 `demographic`

The target audience for the question. Use `"general"` when the question applies to everyone.

**Standard values:**

| Value | Description |
|---|---|
| `general` | No specific demographic; universal applicability |
| `Common, Age 19-30` | Any gender, young professional cohort |
| `Common, Age 31-50` | Any gender, mid-career cohort |
| `Common, Age 50+` | Any gender, senior cohort |
| `Men, Age 19-30` | Men in the young professional cohort |
| `Women, Age 19-30` | Women in the young professional cohort |
| `Men, Age 30-50` | Men in the mid-career cohort |
| `Women, Age 30-50` | Women in the mid-career cohort |
| `Men, Age 50+` | Men, senior cohort |
| `Women, Age 50+` | Women, senior cohort |

For cross-demographic interaction scenarios use the `A vs B` convention:
`"Men 19-30 vs Women 30-50"` — a question about interaction between these two groups.

Keep values consistent across all records for the same country. Inconsistent casing or spacing prevents correct filtering in the admin UI.

---

### 3.5 `lang`

The BCP 47 language tag of the **question text and option text**.

| Tag | Use |
|---|---|
| `en` | English — default for content authored in English |
| `nl` | Dutch / Flemish — for questions authored in Dutch |
| `fr` | French / Walloon — for questions authored in French |
| `de` | German — for questions authored in German (e.g., German-speaking Belgium) |

**Important:** `lang` describes the language of the content, not the region. A question about Walloon etiquette written in English should be `lang: "en"`, not `"fr"`. A question written in French should be `lang: "fr"` regardless of which region it covers.

---

### 3.6 `answer_tier`

Every option must carry an `answer_tier` integer that the learning engine uses to score responses.

| Tier | Label | Meaning |
|---|---|---|
| 1 | **Good** | The ideal, distinguished response. Shows mastery. |
| 2 | **Slightly different** | Acceptable but not optimal. Shows awareness but room to grow. |
| 3 | **Would not do that** | An etiquette mistake. The learner needs correction. |

**Rules:**
- Every question must have **exactly one** tier-1 option.
- The remaining options should be tier 2 and/or tier 3.
- Questions with only tier-2 and tier-3 options (no correct answer) are invalid and will fail validation.

## 4. Example records

### 4.1 Universal English question (Phase 1, Level 1)

```json
{
  "register": "middle-class",
  "research_pillar": "adaptive_linguistics",
  "phase": 1,
  "level": 1,
  "region_code": "BE",
  "demographic": "general",
  "question_text": "You are emailing a potential collaborator for the first time. Which greeting best balances respect and approachability?",
  "historical_context": "Reflects the shift from Victorian formality to merit-based etiquette, where communication became a tool for partnership rather than a display of rank.",
  "options": [
    {
      "text": "\"To whom it may concern, I hereby request a moment of your time.\"",
      "answer_tier": 3,
      "motivation": "This feels overly bureaucratic and creates too much distance."
    },
    {
      "text": "\"Hi [Name], I'm reaching out because I saw your work and wanted to chat.\"",
      "answer_tier": 2,
      "motivation": "Friendly, but starting with 'Hi' and 'chat' for a first-time contact can be too informal."
    },
    {
      "text": "\"Dear [Name], I hope this email finds you well. I am writing to discuss a potential collaboration.\"",
      "answer_tier": 1,
      "motivation": "This is the gold standard. It uses a formal anchor but follows with a warm, professional purpose."
    }
  ],
  "lang": "en"
}
```

## 5. Content rules for new countries

### 5.1 What changes per country

| Element | Universal | Local (per country) |
|---|---|---|
| Phase structure (1–5) | ✓ | |
| Level names (1–5) | ✓ | |
| Demographic bracket values | ✓ | |
| Question format (options + motivation + tier) | ✓ | |
| `research_pillar` names | ✓ | |
| Historical context content | | ✓ |
| Cultural references and examples | | ✓ |
| Phase 4 modules (Regional, Institutional, Milestones) | Template only | ✓ fully local |
| Phase 5 modules (Digital, Multicultural, Class) | Template only | ✓ local thresholds |

### 5.2 Minimum dataset for launch

| Phase | Minimum questions | Required |
|---|---|---|
| Phase 1 — Individual Core | 350 | Yes |
| Phase 2 — Interaction Dynamics | 600 | Yes |
| Phase 3 — Social Navigation | 500 | Yes |
| Phase 4 — Country Layer | 500 | After launch |
| Phase 5 — Contemporary Context | 350 | Optional |

### 5.3 Research pillars

Use consistent snake_case values for `research_pillar`:

| Pillar | `research_pillar` value |
|---|---|
| Adaptive Linguistics | `adaptive_linguistics` |
| Professional Branding | `professional_branding` |
| Social Navigation | `social_navigation` |
| Material Modesty | `material_modesty` |

## 6. Idempotency and re-runs

The seeder uses `onConflictDoNothing()` against a computed hash column:

```
question_hash = md5(region_code | register | research_pillar | phase | level | demographic | lang | question_text)
```

**Consequence:** If you run the seeder twice, no duplicates are inserted. Records are skipped, not updated. To update an existing question's options or historical_context, you must first delete the old record from the database.

**To re-run Belgium:**
```bash
pnpm --filter @workspace/db seed:be
```

---

## 7. Adding a new country

1. **Prepare source material** in any format (Word, Excel, Markdown).
2. **Create `scripts/import-<cc>-learning-tracks.mjs`** following the Belgium script as the template.
3. **Run the script:** `node scripts/import-<cc>-learning-tracks.mjs` → writes `data/<cc>-learning-tracks.json`.
4. **Add an npm script** to `lib/db/package.json`:
   ```json
   "seed:<cc>": "tsx src/seed-learning-tracks.ts ../../data/<cc>-learning-tracks.json"
   ```
5. **Run the seeder:** `pnpm --filter @workspace/db seed:<cc>`.
6. **Verify** in the admin UI: `/admin/learning-tracks?region_code=<CC>`.

## 8. Recommended source format (Excel or Markdown)

When commissioning content from a new country, provide this structure to the content team:

### Excel template (preferred for large datasets)

| Column | Required | Notes |
|---|---|---|
| `register` | Yes | `middle-class` / `professional` / `aristocracy` |
| `phase` | Yes | 1–5 |
| `level` | Yes | 1–5 |
| `demographic` | Yes | Use values from § 3.4 |
| `lang` | Yes | `en` / `nl` / `fr` / `de` |
| `question_text` | Yes | The question |
| `research_pillar` | No | snake_case from § 5.3 |
| `historical_context` | No | 1–3 sentences of historical background |
| `option_a_text` | Yes | Text of option A |
| `option_a_tier` | Yes | 1 / 2 / 3 |
| `option_a_motivation` | Yes | Why this tier |
| `option_b_text` | Yes | Text of option B |
| `option_b_tier` | Yes | 1 / 2 / 3 |
| `option_b_motivation` | Yes | Why this tier |
| `option_c_text` | Yes | Text of option C |
| `option_c_tier` | Yes | 1 / 2 / 3 |
| `option_c_motivation` | Yes | Why this tier |

### Markdown template (for smaller datasets or editorial review)

```markdown
## Phase [N] · Level [N] · [Demographic] · [lang]

**Register:** middle-class

**Question:** [Question text]

**A)** [Option A text]
**Tier:** 1 / 2 / 3
**Motivation:** [Motivation text]

**B)** [Option B text]
**Tier:** 1 / 2 / 3
**Motivation:** [Motivation text]

**C)** [Option C text]
**Tier:** 1 / 2 / 3
**Motivation:** [Motivation text]

**Historical Context:** [1–3 sentences]
```
